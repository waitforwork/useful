обновление k8s с версии 1.30 до 1.31
==========================================
1 сначала обновим узел управления(control-plane) node-01
	1.1 добавление репозитория kubernets 1.31
		curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.31/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring-1.31.gpg
		echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring-1.31.gpg] https://pkgs.k8s.io/core:/stable:/v1.31/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes-1.31.list
		apt-get update
	1.2 обновление kubeadm на node-01
		apt-mark unhold kubeadm && apt-get update && apt-get install -y kubeadm=1.31.0-1.1 && apt-mark hold kubeadm
	1.3 применение обновления control-plane
		kubeadm upgrade apply v1.31.0 --ignore-preflight-errors=CoreDNSUnsupportedPlugins,CoreDNSMigration
	1.4 обновление kubelet и kubectl
		apt-mark unhold kubelet kubectl && apt-get update && apt-get install -y kubelet=1.31.0-1.1 kubectl=1.31.0-1.1 && apt-mark hold kubelet kubectl
	1.5 перезапуск служб
		systemctl daemon-reload && systemctl restart kubelet
2 обновление рабочих узлов node-02
	2.1 добавление репозитория kubernets 1.31
		sudo -i
		curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.31/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring-1.31.gpg
		echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring-1.31.gpg] https://pkgs.k8s.io/core:/stable:/v1.31/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes-1.31.list
		apt-get update
	2.2 обновление kubeadm на node-02
		apt-mark unhold kubeadm && apt-get update && apt-get install -y kubeadm=1.31.0-1.1 && apt-mark hold kubeadm
	2.3 обновление kubelet и kubectl
		apt-mark unhold kubelet kubectl && apt-get update && apt-get install -y kubelet=1.31.0-1.1 kubectl=1.31.0-1.1 && apt-mark hold kubelet kubectl
	2.4 обновить ноду
		kubeadm upgrade node
	2.5 обновление kubelet и kubectl
		sudo apt-mark unhold kubelet kubectl && sudo apt install -y kubelet=1.31.0-1.1 kubectl=1.31.0-1.1 && sudo apt-mark hold kubelet kubectl
	2.6 перезапуск служб
		systemctl daemon-reload && systemctl restart kubelet
